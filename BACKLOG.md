## Идеи и возможности

<!-- Формат: - **Название**: Описание -->

- **Сохранение состояния UI между сессиями**: ✅ (Выполнено 13.04.2026)
Сохранять состояние элементов интерфейса в `localStorage` — например, свернут/развернут раздел завершённых задач (`isCompletedCollapsed` в `Tasks.tsx`), активная вкладка в настройках и т.д. При следующем входе пользователь видит тот же интерфейс, что и при выходе.

- **Обозначение контекста в списке задач**: ✅ (Выполнено 13.04.2026)
 В карточке задачи (`Tasks.tsx`) отображать привязанный контекст — цветной бейдж/точка с названием контекста рядом с заголовком задачи.

- **Палитра цветов для контекстов** ✅ (Выполнено 12.04.2026)
При создании и редактировании контекста заменить текстовое поле `color` на визуальную палитру (color picker) для удобного выбора цвета. 

- **Страница проекта с задачами**: ✅ (Выполнено 12.04.2026)
Возможность перейти в конкретный проект и увидеть только задачи этого проекта — отдельная страница/роут с фильтрацией задач по проекту.

- **Проект в карточке задачи**: ✅ (Выполнено 13.04.2026)
В карточке задачи (`TaskListView`) отображать название проекта, если задача привязана к проекту. Название должно быть кликабельным — при клике переход на страницу проекта (`/projects/:id`).

- **Видимость активных фильтров**: Показывать индикатор/бейдж когда фильтры активны (контекст, область, проект, тег, поиск, сортировка). Добавить кнопку сброса всех фильтров одним кликом. Сейчас фильтры могут оставаться в `localStorage` и применяются при загрузке страницы, но пользователь не видит что они активны и не может быстро их сбросить.

- **Автоочистка корзины**: Автоматически удалять задачи из корзины (trash) через 30 дней после удаления. Реализовать через cron-задачу или механизм очистки при старте приложения.

- **Кнопка очистки корзины**: Добавить кнопку на странице корзины для удаления всех задач одним кликом с подтверждением действия.

- **Экспорт и импорт задач**: Добавить возможность экспортировать все задачи в JSON формат и импортировать их обратно. Функция должна сохранять все связанные данные (проекты, контексты, теги, связи между задачами).
