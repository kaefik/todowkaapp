# TodowkaApp — 4 итерации инкрементальной разработки

Ниже — план, разбитый на 4 итерации. После каждой итерации приложение **полностью работоспособно** и им можно пользоваться «как есть».

---

## Итерация 1 — «Базовый Todo-список»
**Цель:** минимально рабочее приложение — можно регистрироваться, входить и управлять задачами.

| Область | Что делаем |
|---|---|
| **Инфраструктура** | Vite + React SPA, FastAPI, SQLite, Docker-контейнер, базовый CI |
| **Авторизация** | Регистрация (с guard от открытой регистрации), логин/логаут, JWT-токены |
| **Задачи (CRUD)** | Создание, просмотр списка, редактирование, удаление задач |
| **Модель данных** | Таблицы `users`, `tasks` с SQL-индексами |
| **UI** | Простой адаптивный (mobile responsive) интерфейс: список задач, форма добавления, отметка «выполнено» |
| **PWA** | Service Worker, manifest.json — приложение можно «установить» на телефон |

**Результат:** работающий персональный Todo-лист с авторизацией, доступный как PWA.

---

## Итерация 2 — «GTD-методология»
**Цель:** приложение становится полноценным GTD-инструментом.

| Область | Что делаем |
|---|---|
| **Inbox** | Быстрый захват задач (quick capture) |
| **GTD-статусы** | Workflow: Inbox → Active → Waiting → Someday → Completed |
| **Организация** | Контексты, области (areas), теги — полный CRUD |
| **Проекты** | Создание проектов, привязка задач, прогресс-бар |
| **Подзадачи** | Вложенные задачи (subtasks) внутри задачи/проекта |
| **Фильтрация и поиск** | Фильтр по статусу, контексту, тегу; полнотекстовый поиск; сортировка |
| **UI** | Боковая панель с GTD-представлениями (Inbox, Next Actions, Waiting, Someday/Maybe, Projects) |

**Результат:** полноценный GTD-менеджер задач с проектами, контекстами, тегами и подзадачами.

---

## Итерация 3 — «Повторяющиеся задачи, напоминания и синхронизация»
**Цель:** приложение становится «умным» — напоминает, повторяет, синхронизирует в реальном времени.

| Область | Что делаем |
|---|---|
| **Повторяющиеся задачи** | Настройка рекурренсов (ежедневно, еженедельно, custom); APScheduler in-process (через FastAPI lifespan) |
| **Напоминания** | Уведомления о приближающемся дедлайне; job `send_due_reminders` каждую минуту |
| **SSE real-time** | Server-Sent Events для мгновенных уведомлений + SSE-triggered sync между устройствами (push SSE **после** DB commit) |
| **Offline-sync** | Оффлайн-работа через PWA + корректная синхронизация при восстановлении сети (sync fixes) |
| **UI** | Колокольчик уведомлений, панель напоминаний, индикатор онлайн/офлайн |

**Результат:** приложение работает офлайн, уведомляет о задачах и синхронизируется между устройствами в реальном времени.

---

## Итерация 4 — «Еженедельный обзор и управление сессиями»
**Цель:** приложение получает продвинутые GTD-практики и повышенную безопасность.

| Область | Что делаем |
|---|---|
| **Weekly Review** | Страница `/review` — пошаговый wizard: 1) Обработка Inbox до нуля, 2) Проверка активных проектов (есть ли next action?), 3) Ревизия Someday/Maybe (продвинуть или архивировать) |
| **Управление сессиями** | Просмотр активных сессий (устройство, дата, IP), отзыв отдельных сессий, «выйти со всех устройств» |
| **Polish & UX** | Анимации, улучшенная доступность (a11y), dark mode, онбординг для новых пользователей |
| **Hardening** | Rate limiting, аудит безопасности, финальная оптимизация SQL-индексов, нагрузочное тестирование |

**Результат:** зрелое GTD-приложение с еженедельным обзором, управлением сессиями и отполированным UX — готово к повседневному использованию.
