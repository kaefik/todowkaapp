# План: Local-first офлайн-режим на Dexie.js

**Дата:** 2026-04-18
**Статус:** Планирование
**Выбранный вариант:** Вариант 3 — Глубокий рефакторинг (local-first на IndexedDB)

## Постановка проблемы

Приложение должно полноценно работать при отсутствии подключения к бекенду. Текущее состояние:
- Редактирование задачи (PUT) — работает офлайн (благодаря `updateTaskMutation.onMutate` + `localTaskChanges.ts` + IndexedDB)
- Добавление задачи — НЕ работает (нет `onMutate`, задача теряется визуально)
- Toggle задачи — НЕ работает (нет оптимистичного обновления)
- Перемещение задачи — НЕ работает
- Удаление задачи — НЕ работает
- Непосещённые разделы — пустые (данные загружаются только при навигации)
- Справочники (контексты, области, теги, проекты) — только чтение через кэш React Query

## Архитектурные решения

### Подход: Local-first

```
Dexie.js (IndexedDB) ← источник истины
   ↕ useLiveQuery() — реактивные подписки в хуках
   ↕ SyncEngine — фоновая синхронизация с бекендом
Backend API — только приём/отдача данных
```

### Ключевые решения
- **Dexie.js** — локальная база данных на IndexedDB
- **Убрать React Query** — хуки подписываются на Dexie напрямую через `useLiveQuery`
- **Field-merge разрешение конфликтов** — по полям, LWW на уровне каждого поля
- **Итеративная реализация** — сначала задачи, потом справочники

## Field-merge стратегия разрешения конфликтов

Для каждого поля задачи сравниваем `updatedAt`:
- Если локальное изменение новее серверного — берём локальное
- Если серверное новее — берём серверное
- Если оба изменены — по timestamps (LWW на уровне поля)

Структура `localChanges`:
```
{
  taskId: string,
  field: string,        // имя поля (title, description, gtd_status, ...)
  oldValue: any,        // значение до изменения
  newValue: any,        // новое значение
  timestamp: number     // Date.now()
}
```
