## 16. Refresh token — нет автоматического refresh при загрузке

**Задача #23, #20** | Приоритет: 🔴 | Сложность: **Средняя**

**План:** При загрузке приложения — автоматический вызов `/refresh` для восстановления сессии. Access token хранить ТОЛЬКО в памяти.

**Реальность:** `AuthInitializer` вызывает `fetchCurrentUser()`, который использует `GET /api/auth/me` с access token. Но access token хранится в Zustand (память) и теряется при перезагрузке страницы. Механизм:
1. При перезагрузке — `AuthInitializer` вызывает `fetchCurrentUser()`
2. `fetchCurrentUser()` шлёт `GET /api/auth/me` С access token (который null после перезагрузки)
3. Получает 401
4. `httpClient` interceptor пытается `refreshToken()` — вот тут refresh token из cookie и используется
5. Если refresh успешен — пользователь остаётся залогиненным

Это работает, но хрупко: если refresh token тоже истёк — пользователь видит редирект на login без объяснения. Нет UI-индикации «сессия истекла».

**Объём работы:** 1–2 часа — улучшить UX при истёкшей сессии.
