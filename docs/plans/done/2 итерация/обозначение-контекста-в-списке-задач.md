# План реализации: Обозначение контекста в списке задач

---

## Описание фичи

В карточке задачи отображать привязанный контекст — цветной бейдж/точка с названием контекста рядом с заголовком задачи, аналогично тому, как сейчас отображается проект.

**Статус в BACKLOG.md:** Не реализовано
**Приоритет:** 🔴 Высокий (улучшает UX, контекст — ключевая сущность GTD)

---

## Текущее состояние

### Бэкенд
- Модель `Task` имеет поле `context_id` (FK → contexts.id)
- Модель `Context` существует с полями: `id`, `name`, `color`, `icon`
- `TaskResponse` содержит только `context_id: UUID | null` — нет объекта `context` с названием и цветом
- Аналогично, `ProjectBriefResponse` существует и используется в `TaskResponse.project`

### Фронтенд
- Тип `Task` в `useTasks.ts` содержит `context_id: string | null` — нет объекта `context`
- В `TaskListView.tsx` проект отображается (строки 341-353) с цветной точкой и кликабельным названием
- Контекст в списке задач НЕ отображается

### 3. Бэкенд — Обновление API endpoint

**Файл:** `backend/app/api/tasks.py`

**Задача:** Убедиться, что при загрузке задач eagerly загружается связь `context` через SQLAlchemy relationship.

```python
# В запросе добавить eager loading для context
from sqlalchemy.orm import selectinload

# Пример:
stmt = (
    select(Task)
    .options(
        selectinload(Task.project),
        selectinload(Task.context),  # <-- Добавить это
        selectinload(Task.tags)
    )
    .where(...)
)
```

**Причина:** Без eager loading будет N+1 запросов при загрузке списка задач с контекстами.

### 5. Фронтенд — Отображение контекста в TaskListView

**Файл:** `frontend/src/components/TaskListView.tsx`

**Место:** После блока отображения проекта (строки 341-353), добавить аналогичный блок для контекста.

```tsx
{task.context && (
  <div className="mt-1">
    <span className="inline-flex items-center gap-1 text-[10px] text-gray-600 dark:text-gray-400">
      {task.context.color && (
        <span className="w-2.5 h-2.5 rounded-full" style={{ backgroundColor: task.context.color }} />
      )}
      <span className={task.completed ? 'opacity-60' : ''}>{task.context.name}</span>
    </span>
  </div>
)}
```

**Стилизация:**
- Текст серый (не синий как проект), так как контекст пока не имеет своей страницы
- Если в будущем будет страница `/contexts/:id`, можно сделать кликабельным как проект
- Размер и стиль аналогичны проекту для визуальной согласованности

**Причина:** Пользователь видит контекст задачи без необходимости открывать детальный просмотр.

### 6. Тестирование — Бэкенд

```python
# Добавить тест
async def test_task_includes_context(client, auth_headers, test_context):
    """Проверка, что TaskResponse включает объект context"""
    response = await client.get("/api/tasks", headers=auth_headers)
    assert response.status_code == 200
    task = response.json()["items"][0]
    assert task["context"]["id"] == test_context["id"]
    assert task["context"]["name"] == test_context["name"]
    assert task["context"]["color"] == test_context["color"]
```

| Риск | Митигация |
|------|-----------|
