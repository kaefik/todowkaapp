### Lens 3: Assumptions & Risks

**Скрытые предположения:**

1. "Vite proxy поддерживает SSE (EventSource)"
   - **Инверсия:** Vite proxy НЕ поддерживает SSE
   - **Влияние:** SSE не будет работать в dev режиме
   - **Митигация:** Проверить документацию Vite, добавить fallback с прямым подключением или query parameter

2. "Автоматическая миграция IndexedDB сработает без ошибок"
   - **Инверсия:** Миграция частично завершится и сломает кэш
   - **Влияние:** Приложение перестанет работать
   - **Митигация:** Добавить автоматический сброс кэша при критических ошибках с логированием

3. "После исправления cookie secure все 401 ошибки исчезнут"
   - **Инверсия:** Есть другие причины 401 (невалидный токен, не передается, бэкенд не принимает)
   - **Влияние:** Проблемы останутся
   - **Митигация:** Добавить расширенную диагностику (логи токена на клиенте и бэкенде)

### Lens 5: Technical Feasibility

**ПРОБЛЕМА:**
- 🔴 EventSource НЕ поддерживает headers → Vite proxy может не проксировать SSE корректно
- SSE — это long-lived connection, Vite proxy предназначен для обычных HTTP запросов

**Необходимо проверить:**
1. Документацию Vite proxy о поддержке SSE
2. Работает ли SSE через proxy в текущей конфигурации
3. Если нет — реализовать fallback решение
