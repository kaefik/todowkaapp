# Чеклист для ручного тестирования офлайн-режима (Этап 8)

## 1. Тестирование офлайн режима

### 1.1. Отключение бэкенда
- [ ] Остановить бэкенд сервер (`cd backend && ./run.sh` или Ctrl+C)
- [ ] Обновить страницу браузера (F5 или Cmd+R)
- [ ] Приложение должно загрузиться без ошибок

### 1.2. Проверка отображения кэшированных данных
- [ ] Если данные были кэшированы ранее, они должны отображаться
- [ ] При попытке загрузки новых данных должен показываться OfflineBanner
- [ ] Toast уведомление "Вы офлайн" должно появиться
- [ ] Данные должны быть доступны для просмотра

### 1.3. Офлайн мутации
- [ ] Создать новую задачу офлайн
  - Задача должна сохраниться в UI
  - Должен появиться toast "Запрос сохранен и будет отправлен при восстановлении сети"
- [ ] Редактировать задачу офлайн
  - Изменения должны отображаться в UI
  - Toast о сохранении в очередь
- [ ] Удалить задачу офлайн
  - Задача должна исчезнуть из UI
  - Toast о сохранении в очередь
- [ ] Переместить задачу между статусами (GTD) офлайн
  - Изменения должны отображаться
  - Toast о сохранении в очередь

### 1.4. Восстановление сети
- [ ] Запустить бэкенд сервер
- [ ] Подождать автоматической синхронизации (1-5 секунд)
- [ ] Toast "Сеть восстановлена" должен появиться
- [ ] Офлайн мутации должны быть синхронизированы с сервером
- [ ] Данные должны обновиться на сервере
- [ ] OfflineBanner должен исчезнуть

## 2. Тестирование кэша

### 2.1. IndexedDB проверка (в DevTools)
- [ ] Открыть DevTools (F12)
- [ ] Перейти в Application → Storage → IndexedDB
- [ ] Проверить базу данных `todowka-data-cache`
  - [ ] ObjectStore `cache` должен существовать
  - [ ] Данные GET запросов должны быть сохранены
- [ ] Проверить базу данных `todowka-query-cache`
  - [ ] ObjectStore `cache` должен существовать
  - [ ] Query кэш React Query должен быть сохранен
- [ ] Проверить базу данных `todowka-offline-queue` (из Этапа 7)
  - [ ] Очередь мутаций должна существовать

### 2.2. Проверка очистки кэша
- [ ] Выполнить несколько GET запросов (tasks, contexts, areas, projects, tags)
- [ ] Проверить что данные сохранены в IndexedDB
- [ ] Ждать более 24 часов или изменить CACHE_MAX_AGE для теста
- [ ] Перезагрузить приложение
- [ ] Устаревшие записи должны быть удалены из кэша

### 2.3. Проверка лимита размера
- [ ] Выполнить более 100 различных GET запросов
- [ ] Проверить что в кэше только 100 последних записей
- [ ] Старые записи должны быть удалены автоматически

## 3. Оптимизация производительности

### 3.1. Размер IndexedDB
- [ ] Открыть DevTools → Application → Storage → IndexedDB
- [ ] Проверить размер каждой базы данных:
  - [ ] `todowka-data-cache`: < 5MB
  - [ ] `todowka-query-cache`: < 10MB
  - [ ] `todowka-offline-queue`: < 1MB
- [ ] Общий размер должен быть < 50MB

### 3.2. Проверка лишних данных в кэше
- [ ] Просмотреть записи в `todowka-data-cache`
- [ ] Убедиться что только GET запросы кэшируются
- [ ] Нет дубликатов по URL

### 3.3. Проверка времени загрузки
- [ ] Очистить кэш (Application → Storage → Clear site data)
- [ ] Обновить страницу
- [ ] Измерить время первой загрузки (Network tab в DevTools)
- [ ] Загрузить страницу снова (с кэшем)
- [ ] Вторая загрузка должна быть быстрее

## 4. Edge cases

### 4.1. Частое включение/выключение сети
- [ ] Отключить сеть (в Chrome DevTools → Network → Offline)
- [ ] Подождать 5 секунд
- [ ] Включить сеть
- [ ] Повторить 10 раз быстро
- [ ] Приложение не должно вылетать
- [ ] Toast уведомления должны корректно отображаться

### 4.2. Медленное соединение
- [ ] В Chrome DevTools → Network выбрать "Slow 3G"
- [ ] Попробовать загрузить данные
- [ ] Кэшированные данные должны загрузиться быстро
- [ ] OfflineBanner должен показываться при таймауте

### 4.3. Частичные сбои
- [ ] Имитировать сбой только для `/api/tasks` (в DevTools Throttling)
- [ ] Другие API должны работать
- [ ] Задачи должны загружаться из кэша
- [ ] Остальные данные должны загружаться из сети

## 5. Дополнительные проверки

### 5.1. Service Worker
- [ ] Открыть DevTools → Application → Service Workers
- [ ] Service Worker должен быть активным
- [ ] Статус: "Running" или "Activated"
- [ ] Проверить что sw.ts загружен корректно
- [ ] В Console не должно быть ошибок Service Worker

### 5.2. React Query DevTools
- [ ] Открыть React Query DevTools (нажать на значок в левом нижнем углу)
- [ ] Проверить что queries отображаются корректно
- [ ] Проверить что staleTime и gcTime настроены правильно
- [ ] Проверить что mutations работают

### 5.3. PWA установка
- [ ] Открыть приложение в Chrome
- [ ] Проверить что иконка установки PWA доступна (в адресной строке)
- [ ] Установить PWA
- [ ] Приложение должно работать как отдельное окно
- [ ] Офлайн режим должен работать в установленном PWA

## 6. Проверка после всех тестов

### 6.1. Финальная проверка TypeScript
- [ ] `npm run build` не должно иметь ошибок в новых файлах

### 6.2. Финальная проверка Dev Server
- [ ] `npm run dev` запускается без ошибок

### 6.3. Очистка тестовых данных
- [ ] Очистить IndexedDB (Application → Storage → Clear site data)
- [ ] Очистить localStorage (Application → Local Storage → Clear all)
- [ ] Обновить страницу
- [ ] Приложение должно загрузиться и работать корректно

## Примечания для тестировщика

1. Для ускорения тестирования кэша можно изменить `CACHE_MAX_AGE` в `indexedDB.ts` на меньшее значение (например, 1 минуту)
2. Для проверки лимита размера можно изменить `MAX_CACHE_SIZE` на 5-10 записей
3. Для тестирования офлайн мутаций можно посмотреть в DevTools → Application → IndexedDB → todowka-offline-queue
4. При возникновении проблем проверьте Console в DevTools для логов
5. React Query DevTools очень полезен для понимания что происходит с кэшем

## Известные ограничения (из плана)

1. **Нет conflict resolution** — при одновременном редактировании используется last write wins
2. **Нет partial updates** — при ошибке синхронизации вся мутация не выполняется
3. **Оптимистичные обновления** — не реализованы (опционально)

Эти ограничения не являются критичными для базовой офлайн работы приложения.
