## 📝 2. Создание тестовых уведомлений

### Способ 1: Прямой SQL в SQLite (рекомендуется)

```bash
# Перейдите в директорию с базой данных
cd backend/data

# Откройте SQLite
sqlite3 todowka.db

# Получите ваш user_id (замените YOUR_USERNAME)
SELECT id FROM users WHERE username = 'YOUR_USERNAME';

# Создайте тестовые уведомления (замените USER_ID на полученный id)
INSERT INTO notifications (id, user_id, type, message, is_read, created_at) VALUES
  ('test-1', 'USER_ID', 'due_reminder', 'Напоминание: задача с дедлайном сегодня', 0, datetime('now')),
  ('test-2', 'USER_ID', 'recurrence_created', 'Создана следующая повторяющаяся задача', 0, datetime('now', '-5 minutes')),
  ('test-3', 'USER_ID', 'task_updated', 'Задача была обновлена', 1, datetime('now', '-10 minutes'));

# Выйдите из SQLite
.exit
```

### Способ 2: Через Python скрипт

Создайте файл `create_test_notifications.py` в директории `backend/`:

```python
import asyncio
from sqlalchemy import select
from app.database import async_session_maker
from app.models.notification import Notification
from app.models.user import User

async def create_test_notifications():
    async with async_session_maker() as session:
        # Получите первого пользователя
        result = await session.execute(select(User).limit(1))
        user = result.scalar_one_or_none()
        
        if not user:
            print("No users found. Please create a user first.")
            return
        
        # Создайте тестовые уведомления
        notifications = [
            Notification(
                user_id=user.id,
                type='due_reminder',
                message='Напоминание: задача с дедлайном сегодня',
                is_read=False
            ),
            Notification(
                user_id=user.id,
                type='recurrence_created',
                message='Создана следующая повторяющаяся задача',
                is_read=False
            ),
            Notification(
                user_id=user.id,
                type='task_updated',
                message='Задача была обновлена',
                is_read=True
            )
        ]
        
        session.add_all(notifications)
        await session.commit()
        print(f"Created {len(notifications)} test notifications for user {user.username}")

if __name__ == '__main__':
    asyncio.run(create_test_notifications())
```

Запустите скрипт:
```bash
cd backend
python create_test_notifications.py
```

## 🔄 6. Тестирование Exponential Backoff

### Шаги:
1. Откройте DevTools (F12) → Network → WS/SSE
2. Найдите подключение к `/api/sse/notifications`
3. Остановите backend (Ctrl+C в терминале)
4. Наблюдайте за поведением в UI

**Что проверить:**
- ✅ SSE подключение закрывается (статус меняется на disconnected/error)
- ✅ Через 1 секунду происходит попытка переподключения
- ✅ Если backend не запущен, следующая попытка через 2 секунды
- ✅ Затем 4, 8, 16, 30 секунд (максимум)
- ✅ Запустите backend снова: `./run.sh`
- ✅ Подключение должно восстановиться автоматически
- ✅ Индикатор подключения возвращается к "connected"

**Визуальная проверка в DevTools:**
```
✓ /api/sse/notifications (pending)
✗ /api/sse/notifications (failed)
✓ /api/sse/notifications (pending) - через 1s
✓ /api/sse/notifications (200 OK) - после восстановления backend
```

## 🐛 7. Диагностика проблем

### Нет уведомлений в UI

**Проверки:**
```bash
# 1. Проверьте что уведомления есть в БД
cd backend/data
sqlite3 todowka.db
SELECT * FROM notifications WHERE user_id = 'YOUR_USER_ID';

# 2. Проверьте что backend запущен
curl http://localhost:8000/health

# 3. Проверьте консоль браузера на ошибки
# F12 → Console
```

### SSE не подключается

**Проверки:**
1. Откройте DevTools (F12) → Network
2. Найдите запрос к `/api/sse/notifications`
3. Проверьте статус ответа (должен быть 200)
4. Проверьте Response tab - должны приходить события каждую секунду

```javascript
// В браузерной консоли для отладки
// Откройте EventSource вкладку в Network
// Или проверьте вручную:
const es = new EventSource('http://localhost:8000/api/sse/notifications');
es.onmessage = (e) => console.log('SSE:', e.data);
```

### Уведомления не обновляются в реальном времени

**Проверки:**
1. DevTools → Network → найдите SSE запрос
2. Перейдите в "Messages" или "EventStream" tab
3. Должны быть события с типом `event: notification`
4. Проверьте консоль на ошибки JavaScript

**Типичные проблемы:**
- ❌ CORS блокирует SSE запросы (проверьте консоль)
- ❌ Бэкенд не запущен или падает с ошибкой
- ❌ Неверный API_BASE_URL в `.env` фронтенда
- ❌ Некорректный токен авторизации

## 🛠 10. Дополнительные инструменты тестирования

### Тестирование SSE endpoint напрямую

```bash
# Тестирование notifications stream
curl -N http://localhost:8000/api/sse/notifications \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

# Тестирование sync stream
curl -N http://localhost:8000/api/sse/sync \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

### Мониторинг базы данных в реальном времени

```bash
cd backend/data
sqlite3 todowka.db

# Мониторинг новых уведомлений
.mode box
.timer on
SELECT * FROM notifications ORDER BY created_at DESC LIMIT 10;
```

### Проверка API уведомлений

```bash
# Получить все уведомления
curl http://localhost:8000/api/notifications \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" | jq

# Отметить как прочитанное
curl -X PATCH http://localhost:8000/api/notifications/NOTIFICATION_ID/read \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

# Отметить все как прочитанные
curl -X PATCH http://localhost:8000/api/notifications/read-all \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

# Удалить уведомление
curl -X DELETE http://localhost:8000/api/notifications/NOTIFICATION_ID \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## 🎯 Критерии успеха тестирования

Тестирование считается успешным, если:

1. **UI работает:**
   - Все компоненты уведомлений отображаются корректно
   - Взаимодействие интуитивно понятно
   - Стили соответствуют дизайн-системе

2. **Real-time работает:**
   - Уведомления появляются мгновенно (< 1 секунда)
   - Синхронизация между браузерами работает без задержек
   - SSE соединение стабильно

3. **Надежность:**
   - Exponential backoff работает корректно
   - Переподключение происходит автоматически
   - Падение бэкенда не ломает UI

4. **Производительность:**
   - Уведомления не тормозят UI
   - SSE потребляет минимум ресурсов
   - Нет утечек памяти при долгой работе
