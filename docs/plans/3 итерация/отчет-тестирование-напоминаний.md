### Критические проблемы

#### 1. **Задачи с last_reminder_sent_at, но без уведомлений** 🔴
**Затронуто:** 2 задачи

**Задача "Тест полной цикл 12:36:48" (ID: 8d34d555-4b5a-4f6c-9323-1f42aba77168)**
```
Due date: 2026-04-14 12:41:48 UTC
Reminder time: 15:36:48
Last reminder sent: 2026-04-14 12:36:48 ✅
Уведомления: НЕТ ❌
```

**Задача "Тест 15:22:52" (ID: dd39feea-c217-4344-b489-80851951c3f1)**
```
Due date: 2026-04-14 00:00:00 UTC
Reminder time: 15:58:00
Last reminder sent: 2026-04-14 12:23:02 ✅
Уведомления: НЕТ ❌
```

**Анализ:**
- Планировщик запустился и установил `last_reminder_sent_at`
- НО уведомление НЕ было создано
- Уведомления в ±10 секунд от этих моментов отсутствуют
- Это указывает на **ошибку в send_reminder() или проблему с транзакцией**

**Вероятная причина:** Проблема в `reminder_service.py:89-90`:
```python
task.last_reminder_sent_at = datetime.now(ZoneInfo('UTC'))
await self.db.commit()  # Коммит ВНУТРИ send_reminder
```

Затем в `scheduler.py:119` еще один коммит:
```python
await session.commit()  # Двойной коммит!
```

Это может привести к race conditions или частичным коммитам.

### Средние проблемы

#### 3. **Некорректное хранение reminder_offsets** 🟡
**Проблема:** В БД `reminder_offsets` хранится как строка `"null"` вместо `null` или `[]`

**Данные из БД:**
```sql
reminder_offsets (raw): "null"
reminder_offsets (type): <class 'str'>
reminder_offsets (parsed): None
```

**Влияние:** При попытке использовать `timedelta(minutes=offset_minutes)` возникнет `TypeError`, так как offset_minutes будет строкой.

---

#### 4. **Задачи без действующих напоминаний** 🟡
**Затронуто:** 1 задача

**Задача "Задача с напоминанием" (ID: ab3d91fb-c4f3-4312-9e9a-d3d5bd63759b)**
```
Due date: 2026-04-15 12:00:00 UTC
Reminder time: None ❌
Reminder offsets: "null" ❌
```

**Проблема:** Задача помечена как "с напоминанием", но не имеет ни `reminder_time`, ни валидных `reminder_offsets`.

---

#### 5. **Сериализация reminder_time с микросекундами** 🟡
**Проблема:** В БД `reminder_time` хранится с микросекундами:
```sql
reminder_time: 17:00:00.000000
```

**Влияние:** Фронтенд ожидает формат `HH:MM`, получает `HH:MM:SS.ffffff`. Pydantic должен обрабатывать это, но есть риск проблем при парсинге.

## 🔍 Детальный анализ критических проблем

### Почему задачи "Тест полной цикл 12:36:48" и "Тест 15:22:52" не получили уведомления?

**Гипотеза 1: Ошибка в send_reminder()**
```python
# reminder_service.py:68-92
async def send_reminder(self, task: Task, user: User) -> Notification:
    ...
    notification = await self.create_notification(...)  # Шаг 1
    task.last_reminder_sent_at = datetime.now(ZoneInfo('UTC'))  # Шаг 2
    await self.db.commit()  # Шаг 3
    return notification
```

Если между Шагом 1 и Шагом 3 происходит исключение:
- `notification` создается в памяти
- `last_reminder_sent_at` устанавливается
- `commit()` не выполняется → откат транзакции
- НО `last_reminder_sent_at` может остаться в сессии

**Гипотеза 2: Проблема с двойным коммитом**
```python
# scheduler.py:117-119
if user:
    notification = await reminder_service.send_reminder(task, user)  # Коммит #1
    logger.info(f"Sent reminder...")
    await session.commit()  # Коммит #2 - лишний!
```

Двойной коммит может привести к:
- Race conditions
- Частичному сохранению данных
- Потере уведомлений

## 🛠️ Рекомендации по исправлению

### 1. **Исправить проблему с транзакциями (КРИТИЧНО)**

**В reminder_service.py:**
```python
# УБРАТЬ коммит из send_reminder()
async def send_reminder(self, task: Task, user: User) -> Notification:
    ...
    notification = await self.create_notification(...)
    task.last_reminder_sent_at = datetime.now(ZoneInfo('UTC'))
    # await self.db.commit()  # ❌ УБРАТЬ
    await self.db.flush()  # ✅ Использовать flush вместо commit
    return notification
```

**В scheduler.py:**
```python
# Оставить только один коммит
for task in due_tasks:
    try:
        if reminder_service.should_send_reminder(task):
            ...
            if user:
                notification = await reminder_service.send_reminder(task, user)
                logger.info(f"Sent reminder...")
                # await session.commit()  # ❌ УБРАТЬ (коммит будет после цикла)
    except Exception as e:
        logger.error(f"Error sending reminder: {e}")
        await session.rollback()

await session.commit()  # ✅ Один коммит для всех задач
```
