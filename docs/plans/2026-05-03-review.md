# Insight-Driven Fast Review (Вариант 1+2+3)

Дата: 2026-05-03
Статус: Approved

## Суть

Объединение трёх подходов: починка фундамента (1) → дашборд контекста (2) → быстрый триаж (3). Принцип: сначала контекст, потом действие, без лишних шагов.

## Архитектурные решения

### Layout: Fullscreen (вариант A)

Ревью остаётся полноэкранным (вне AppLayout), как сейчас. Текущий минимап-сайдбар (220px) убирается, заменяется на тонкий progress bar вверху.

Роутинг не меняется: `/review` — топ-level route, вне AppLayout.

**Почему не in-layout:** Ревью — ритуал, нужен фокус. Полноэкранный режим создаёт нужный контекст переключения.

### completed_at: уже существует

Поле `completed_at` уже есть на модели Task и устанавливается при завершении. Данные для "done за неделю" доступны. Не требует изменений.

## Общий флоу

```
Dashboard (insights)  →  Triage Inbox (swipe)  →  Projects (fix alerts)  →  Someday (swipe)  →  Done
        ↑                         ↑                       ↑                        ↑
   "Что происходит?"        "Быстро решай"          "Где проблемы?"         "Что активировать?"
```

Навигация нелинейная: из Dashboard можно перейти к любому шагу. После обработки раздела — возврат на Dashboard.

## Шаг 0: Dashboard (вход в ревью)

Новая стартовая точка. Заменяет текущий вход "сразу в Inbox".

```
┌─────────────────────────────────────────────────────────┐
│  ← Exit                               Weekly Review #14 │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐   │
│  │   12     │ │    3     │ │    7     │ │    2     │   │
│  │  inbox   │ │ overdue  │ │  done ↑  │ │  stale   │   │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘   │
│                                                         │
│  ┌─── Week Activity ─────────────────────────────────┐  │
│  │  Mon  Tue  Wed  Thu  Fri  Sat  Sun               │  │
│  │   2    1    3    0    1    -    -    total: 7 done │  │
│  └───────────────────────────────────────────────────┘  │
│                                                         │
│  ⚠️ Требуют внимания:                                  │
│  ┌──────────────────────────────────────────────────┐   │
│  │ 🔴 Website Redesign — нет next action (14 дней)  │   │
│  │ 🔴 Fitness Plan — нет next action (7 дней)       │   │
│  │ 🟡 5 someday задач без движения > 30 дней        │   │
│  │ 🟡 2 задачи просрочены                            │   │
│  └──────────────────────────────────────────────────┘   │
│                                                         │
│  ┌─── Плашка прогресса ─────────────────────────────┐   │
│  │  ●●●●●●●●●●○○○○○○○○○○  Health: 7/10             │   │
│  │  Лучше чем на прошлой неделе (5/10) ↑             │   │
│  └──────────────────────────────────────────────────┘   │
│                                                         │
│  ┌─────────────┐ ┌──────────────┐ ┌────────────────┐    │
│  │ ▶ Inbox 12  │ │ ▶ Projects 2 │ │ ▶ Someday 15   │    │
│  │   triage    │ │   fix alerts │ │   triage       │    │
│  └─────────────┘ └──────────────┘ └────────────────┘    │
│                                                         │
│           [ Mark Complete (skip triage) ]                │
└─────────────────────────────────────────────────────────┘
```

Ключевые решения:
- 4 карточки-метрики наверху: inbox / overdue / done / stale
  - inbox: `SELECT COUNT(*) WHERE gtd_status = 'inbox'`
  - overdue: `SELECT COUNT(*) WHERE due_date < now() AND gtd_status NOT IN ('completed', 'trash')`
  - done: `SELECT COUNT(*) WHERE completed_at >= :week_start` (поле уже существует)
  - stale: `SELECT COUNT(*) WHERE gtd_status IN ('active', 'next') AND updated_at < now() - interval '14 days'`
- Activity strip — done per day за последние 7 дней (CSS-бары, без chart-библиотеки)
  - Запрос: `SELECT DATE(completed_at), COUNT(*) FROM tasks WHERE completed_at >= :week_start GROUP BY DATE(completed_at)`
- Секция "Требует внимания" — только проблемы, без шума
- Health score (упрощённый, 3 уровня):
  - 🟢 OK — нет проектов без next action, inbox < 10, нет overdue
  - 🟡 Attention — 1-2 проблемы
  - 🔴 Problems — 3+ проблемы или проекты без next action > 3
- 3 кнопки перехода к триажу — можно начинать с любого шага
- "Mark Complete" — зафиксировать ревью без триажа

## Шаг 1: Inbox Triage (swipe-карточки)

Из dashboard нажимаешь "Inbox 12" — переходишь в быстрый триаж.

```
┌─────────────────────────────────────────────┐
│  ← Dashboard   ●●●●●○○○○○○○  5/12 inbox    │
├─────────────────────────────────────────────┤
│                                             │
│          ┌─────────────────────┐            │
│          │                     │            │
│          │  Купить батарейки   │            │
│          │                     │            │
│          └─────────────────────┘            │
│                                             │
│     ▸ Next: Позвонить в страховую          │
│                                             │
├─────────────────────────────────────────────┤
│  [1] Do It    [2] Project    [3] Someday    │
│                    [4] Trash                │
│                    [Skip →]                 │
└─────────────────────────────────────────────┘
```

Отличия от текущего:
- Убран минимап-сайдбар (220px) — заменён на тонкий progress bar сверху
- Карточка по центру, крупнее — фокус на одной задаче
- Кнопки в один ряд (не 2x2): Do It | Project | Someday | Trash
- Skip — задача остаётся в inbox, не помечается как обработанная
- Layout: полноэкранный, без AppLayout sidebar (как сейчас)
- Keyboard shortcuts: 1-4, S для Skip
- Кнопка "← Dashboard" вместо "Cancel"

## Починка багов

| Проблема | Решение |
|----------|---------|
| N+1 запросы | Один запрос с selectinload для проектов |
| Хардкод 7 дней | Везде используется review_frequency_days |
| Нет истории | Новая таблица review_snapshots |
| Нет навигации назад | Кнопка "← Dashboard" на каждом шаге |
| i18n ключи | Все ключи добавлены в переводы |
| Someday без due_date | Добавлен due_date в запрос |
| Banner запрашивает весь статус | Новый endpoint GET /review/summary (только метрики) |
| ReviewReminderBanner хардкодит 7 дней | Использовать review_frequency_days |

## Новая архитектура бэкенда

### Таблица review_snapshots

```sql
CREATE TABLE review_snapshots (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36) REFERENCES users(id),
    created_at DATETIME WITH TIME ZONE DEFAULT NOW(),
    inbox_count INTEGER NOT NULL,
    overdue_count INTEGER NOT NULL,
    done_count INTEGER NOT NULL,
    stale_count INTEGER NOT NULL,
    projects_without_next INTEGER NOT NULL,
    health_score VARCHAR(10) NOT NULL,  -- 'ok', 'attention', 'problems'
    inbox_processed INTEGER DEFAULT 0,
    next_actions_added INTEGER DEFAULT 0,
    someday_activated INTEGER DEFAULT 0
);
```

### Миграция

```bash
cd backend
alembic revision --autogenerate -m "add review_snapshots table"
alembic upgrade head
```

### API endpoints

| Method | Path | Описание | Phase |
|--------|------|----------|-------|
| GET | /review/summary | Метрики для dashboard (лёгкий, без N+1) | MVP |
| GET | /review/status | Полные данные для триажа (inbox/projects/someday) | MVP |
| POST | /review/complete | Сохранить снапшот + обновить дату | MVP |
| GET | /review/history | Последние N снапшотов для сравнения | Phase 2 |

### /review/summary response schema

```python
class ReviewSummaryResponse(BaseModel):
    inbox_count: int
    overdue_count: int
    done_this_week: int
    stale_count: int
    projects_without_next: int
    health_status: Literal['ok', 'attention', 'problems']
    last_review_date: datetime | None
    review_count: int
    week_activity: dict[str, int]  # {"Mon": 2, "Tue": 1, ...}
    alerts: list[ReviewAlert]

class ReviewAlert(BaseModel):
    severity: Literal['red', 'yellow']
    message: str
    project_id: str | None = None
```

## Фазировка

### Phase 1 (MVP) — ~7-9 дней

1. Бэкенд: починка N+1, хардкод 7 дней, someday due_date
2. Бэкенд: таблица review_snapshots + миграция
3. Бэкенд: `GET /review/summary` (лёгкий endpoint)
4. Бэкенд: обновить `POST /review/complete` — сохранять снапшот
5. Фронтенд: ReviewDashboard — метрики + alerts + кнопки навигации
6. Фронтенд: убран ReviewMinimap → progress bar
7. Фронтенд: Inbox/Someday triage — центральная карточка, skip, ← Dashboard
8. Фронтенд: Projects — problem-first, show all toggle
9. Фронтенд: Completion — stats + health до/после
10. i18n: все ключи

### Phase 2 — ~3-5 дней

1. Activity strip (done per day)
2. `GET /review/history` — сравнение с прошлыми неделями
3. Resume progress (автосохранение)
4. ReviewReminderBanner: использовать `/review/summary` вместо `/review/status`

## Оценка

| Критерий | Оценка |
|----------|--------|
| Решает "без удовольствия" | Да — быстрый триаж + контекст = удовлетворение |
| Даёт инсайты | Да — dashboard + alerts + health |
| Ускоряет процесс | Да — swipe + problem-first + skip |
| Сложность MVP | Средняя (~7-9 дней) |
| Риск регрессии | Низкий (новые endpoints, старые не ломаются) |
