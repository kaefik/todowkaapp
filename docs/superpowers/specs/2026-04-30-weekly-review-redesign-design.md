# Weekly Review Redesign: Wizard + Миникарта

**Дата:** 2026-04-30  
**Статус:** На утверждении

## Проблема

Текущая реализация Weekly Review — линейный wizard на 4 шага без общей видимости. Полнофункциональные компоненты ReviewInbox и ReviewSomeday не подключены (шаги показывают просто списки). Пользователь не видит картину целиком — только фрагменты по шагам.

## Решение

**Wizard + Миникарта** — сохранить пошаговый подход, но добавить боковую панель-миникарту (220px) слева, которая показывает все секции с прогрессом и позволяет свободно перемещаться между ними.

## Архитектура

### Layout

```

│   3/5 ████░  │                                 │
```

### Компоненты

1. **`ReviewMinimap`** — левая панель с навигацией
2. **`ReviewWorkspace`** — обёртка рабочей области
3. **`ReviewInboxStep`** — переработанный шаг Inbox (карточка + 4 кнопки)
4. **`ReviewProjectsStep`** — текущий шаг Projects (с минимальными улучшениями)
5. **`ReviewSomedayStep`** — переработанный шаг Someday (карточка + 3 кнопки)
6. **`ReviewCompletion`** — экран завершения со статистикой
7. **`ReviewPage`** — страница-контейнер `/review` (заменяет Review.tsx)

### State management

Zustand-стор `useReviewStore` для:
- `currentStep` — текущая секция (inbox/projects/someday/completion)
- `data` — данные из API (inbox_tasks, active_projects, someday_tasks)
- `progress` — обработано/всего по каждой секции
- `stats` — накопленная статистика для completion

## Детали по шагам

### Шаг 1: Inbox

**Паттерн:** Фокус по одной задаче + 4 кнопки действий.

- Текущая задача показана крупной карточкой (заголовок + описание)
- 4 кнопки действий в сетке 2x2:
  - **Сделать** (клавиша 1) — статус → Next Action
  - **В проект** (клавиша 2) — inline выбор проекта, назначить задачу
  - **Когда-нибудь** (клавиша 3) — статус → Someday/Maybe
  - **Удалить** (клавиша 4) — удалить задачу
- Превью следующих 2 задач внизу (контекст)
- Автопереход к следующей задаче после действия
- Автопереход к Projects когда Inbox пуст
- Индикатор «ЗАДАЧА 3 ИЗ 5»

### Шаг 2: Projects

**Паттерн:** Список проектов с акцентом на проблемные.

- Активные проекты с индикатором (зелёный = есть next action, красный = нет)
- Красная рамка + фон для проектов без next action — привлекают внимание
- Раскрывающаяся форма для добавления next action
- Счётчик задач в проекте
- Отмечается обработанным при просмотре или добавлении next action

### Шаг 3: Someday/Maybe

**Паттерн:** Фокус по одной задаче + 3 кнопки (аналог Inbox).

- Текущая задача + дата добавления (контекст «как давно лежит»)
- 3 кнопки:
  - **Активировать** (клавиша 1) — статус → Inbox
  - **Оставить** (клавиша 2 / Enter) — без изменений
  - **В корзину** (клавиша 3) — удалить
- Превью следующих 2 задач

### Шаг 4: Completion

- Миникарта полностью зелёная (все секции с галочками)
- Центральный контент: большая галочка + «Обзор завершён!»
- 3 карточки статистики:
  - Inbox обработано
  - Next action добавлено
  - Someday активировано
- Номер обзора + дата следующего (через 7 дней)
- Кнопка «На главную» → `/`
- При маунте вызывается `POST /api/review/complete`

## Миникарта: поведение

- **Всегда видима** — зафиксирована слева
- **Секции** — кликабельные, переходят к соответствующему шагу
- **Текущая секция** — выделена бордером и тенью
- **Бейджи** — количество задач в каждой секции
- **Прогресс-бары** — обработано/всего для каждой секции
- **Общий прогресс** — внизу панели, суммарная полоска + текст
- **Предупреждения** — «3 без next action» под Projects
- **Завершённые секции** — зелёная галочка ✓

## Навигация

- Клик по секции в миникарте → переход к шагу
- Кнопки «← Inbox» / «Someday →» в заголовке рабочей области
- Клавиша Escape → отмена, возврат на главную
- Свободный переход между секциями (не обязательно по порядку)
- Можно вернуться к обработанной секции

## Клавиатурные шорткаты

| Клавиша | Действие |
|---------|----------|
| 1-4 | Действия на Inbox-шаге |
| 1-3 | Действия на Someday-шаге |
| Enter | Основное действие (Оставить на Someday, Далее на Projects) |
| Escape | Отмена / На главную |
| ← → | Переключение секций |

## API изменения

**Без изменений на бэкенде.** Существующие эндпоинты:
- `GET /api/review/status` — данные для обзора
- `POST /api/review/complete` — завершение

Данные для прогресса считаются на фронтенде (отслеживание обработанных задач).
