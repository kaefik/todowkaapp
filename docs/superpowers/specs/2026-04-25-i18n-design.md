# Design: Мультиязычный интерфейс (i18n)

**Дата:** 2026-04-25
**Статус:** Approved

## Цель

Добавить поддержку мультиязычности в фронтенд Todowka: русский (RU) и английский (EN) с возможностью добавления новых языков.

## Библиотека

**react-i18next** + **i18next** — стандарт де-факто для React-приложений.

## Архитектура

### Структура файлов

```
frontend/src/i18n/
├── index.ts              # Инициализация i18next
├── locales/
│   ├── ru/
│   │   ├── common.json   # Кнопки, подтверждения, статусы
│   │   ├── nav.json      # Навигация, сайдбар
│   │   ├── auth.json     # Логин, регистрация, timezone
│   │   ├── tasks.json    # Задачи: CRUD, фильтры, даты, GTD
│   │   ├── settings.json # Настройки, профиль, пароль
│   │   ├── projects.json # Проекты, области, контексты, теги
│   │   ├── notifications.json # Уведомления
│   │   ├── sync.json     # Офлайн, синхронизация, ошибки
│   │   └── verbs.json    # Быстрые глаголы
│   └── en/
│       └── (аналогичная структура)
```

### Namespace'ы (9 штук)

| Namespace | Содержимое | ~Строк |
|-----------|-----------|--------|
| common | Кнопки (Сохранить, Отмена, Удалить...), подтверждения, статусы | ~30 |
| nav | Сайдбар, хедер, поиск, GTD-секции | ~25 |
| auth | Логин, регистрация, timezone-setup | ~15 |
| tasks | CRUD задач, GTD-статусы, фильтры, даты, детали, подзадачи, повторения, напоминания | ~120 |
| settings | Настройки (все вкладки), профиль, пароль, пользователи | ~85 |
| projects | Проекты, области, контексты, теги (CRUD) | ~65 |
| notifications | Уведомления, колокольчик, SSE | ~25 |
| sync | Офлайн, синхронизация, ошибки, error boundary | ~25 |
| verbs | Быстрые глаголы | ~15 |

**Итого: ~410 строк**

### Определение и сохранение языка

1. При первом входе: `navigator.language` → если `ru` или `ru-RU`, то RU, иначе EN
2. Сохраняется в `localStorage` (ключ `i18nextLng`) — автоматически i18next
3. В настройках — dropdown с выбором языка
4. Язык НЕ сохраняется на сервере (YAGNI — пока достаточно localStorage)

### Переключение языка

- Вкладка «Общие» в Настройках → секция «Язык» → dropdown RU/EN
- При смене языка — мгновенная перерисовка без перезагрузки

### Форматирование дат

- `toLocaleDateString(locale)` параметризуется через текущий язык
- Склонения (подзадача/подзадачи/подзадач) — через i18next plural suffixes

### Добавление нового языка

1. Создать папку `frontend/src/i18n/locales/<lang>/`
2. Скопировать JSON-файлы из `en/` и перевести
3. Добавить язык в массив `supportedLanguages` в `i18n/index.ts`
4. Добавить опцию в dropdown настроек

## Что НЕ входит

- Backend API не меняется
- Сообщения об ошибках от бэкенда не переводятся
- Push-уведомления браузера не переводятся (пока)
- Тесты обновляются по ходу
