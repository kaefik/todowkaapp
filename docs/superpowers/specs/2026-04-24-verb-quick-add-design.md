# Verb Quick-Add: Быстрые глаголы для добавления задач

## Проблема

Пользователь часто создаёт задачи, начинающиеся с глаголов (купить, сделать, проверить, позвонить и т.д.). Каждый раз вводить глагол вручную — лишние действия. Нужен быстрый способ добавить задачу с предустановленным глаголом без ввода с клавиатуры.

## Решение

Глаголы — это настраиваемый список «быстрых шаблонов». При выборе глагола он предзаполняет поле ввода задачи. Интерфейс адаптивный:

- **Desktop (>=768px):** чипы-глаголы над полем ввода, появляются при фокусе, скрываются при потере фокуса
- **Mobile (<768px):** поле ввода скрыто. FAB-кнопка ✏️ в правом нижнем углу раскрывает веер глаголов. При выборе глагола появляется поле ввода с предзаполненным глаголом.

## Предустановленные глаголы

По умолчанию (если пользователь ещё не настраивал список):

| Глагол | Иконка |
|--------|--------|
| Купить | 🛒 |
| Сделать | 🔨 |
| Проверить | ✅ |
| Позвонить | 📞 |
| Написать | ✉️ |
| Найти | 🔍 |

## Desktop: Чипы по фокусу

### Без фокуса
```
┌─────────────────────────────────┐
│ [Добавьте задачу...      ] [Add] │
└─────────────────────────────────┘
```

### С фокусом
```
┌═══════════════════════════════════════════════════════┐
│ [🛒 Купить] [🔨 Сделать] [✅ Проверить] [📞 Позвонить] │
│ [✉️ Написать] [🔍 Найти] [+ ещё]                      │
│ ───────────────────────────────────────────────────── │
│ [Добавьте задачу...                            ] [Add] │
└═══════════════════════════════════════════════════════┘
```

**Поведение:**
- Клик на чип → глагол вставляется в начало поля ввода, курсор после пробела
- Повторный клик на активный чип → убирает глагол, очищает поле
- Клик на другой чип → заменяет текущий глагол
- Клик «+ ещё» → инлайн-поле для ввода нового глагола
- Потеря фокуса → чипы плавно скрываются

## Mobile: FAB с веером глаголов

### Покой
- Поле ввода **скрыто**. Показывается только список задач.
- FAB-кнопка ✏️ в правом нижнем углу.

### После тапа на FAB
- Раскрывается веер глаголов снизу вверх
- Выбранный глагол подсвечен
- FAB-кнопка поворачивается на 45° (✏️ → ✕)

### После выбора глагола
- Появляется поле ввода с предзаполненным глаголом
- Фокус на продолжение текста
- FAB остаётся открытым для смены глагола

### После добавления задачи
- Поле ввода и веер **полностью скрываются**
- Возврат в состояние покоя

## Управление глаголами

### Добавление своего глагола
- Кнопка «+ ещё» (desktop) / «+ добавить свой» (mobile в веере) → инлайн-поле для ввода текста глагола
- Новый глагол добавляется в конец списка
- Автоматически назначается случайная иконка из набора emoji (пользователь может сменить в настройках)

### Страница настроек
- Отдельная секция «Быстрые глаголы» в настройках профиля
- Список всех глаголов с возможностью:
  - Редактировать текст и иконку
  - Удалить глагол (свайпом или кнопкой)
  - Изменить порядок (drag & drop)
  - Добавить новый
- Кнопка «Сбросить по умолчанию» для возврата к предустановленным

## Хранение и синхронизация

Архитектура offline-first, как у задач: сначала сохраняем локально, потом синхронизируем с сервером.

### Backend

**Модель `verb_templates`** (новая таблица):

| Поле | Тип | Описание |
|------|-----|----------|
| id | String(36) PK | UUID |
| user_id | String(36) FK | Владелец |
| text | String(30) | Текст глагола |
| icon | String(10) | Emoji-иконка |
| position | Integer | Порядок сортировки |
| created_at | DateTime | Создано |
| updated_at | DateTime | Обновлено |

**API endpoints:**

| Метод | Путь | Описание |
|-------|------|----------|
| GET | /verb-templates | Получить список глаголов пользователя |
| POST | /verb-templates | Создать глагол |
| PUT | /verb-templates/{id} | Обновить глагол |
| DELETE | /verb-templates/{id} | Удалить глагол |
| PUT | /verb-templates/reorder | Изменить порядок (массив ID) |
| POST | /verb-templates/reset | Сбросить к предустановленным |

**Pydantic-схемы:**
```python
class VerbTemplateCreate(BaseModel):
    text: str = Field(min_length=1, max_length=30)
    icon: str = Field(min_length=1, max_length=10)

class VerbTemplateUpdate(BaseModel):
    text: str | None = Field(None, min_length=1, max_length=30)
    icon: str | None = Field(None, min_length=1, max_length=10)

class VerbTemplateResponse(BaseModel):
    id: str
    text: str
    icon: str
    position: int
    created_at: datetime
    updated_at: datetime
```

### Frontend (offline-first)

**IndexedDB (Dexie):**
- Таблица `verbTemplates` с полями: `id`, `text`, `icon`, `position`, `_syncStatus`
- Таблица `mutations` — добавляется `action: 'create_verb' | 'update_verb' | 'delete_verb' | 'reorder_verbs'`

**Синхронизация через существующий SyncProvider:**
- При запуске: `GET /verb-templates` → заменить локальные данные серверными
- При мутации: записать в `mutations` → SyncProvider отправит на сервер
- Конфликт: серверная версия побеждает (last-write-wins по `updated_at`)

**Предустановленные глаголы:**
- Если `GET /verb-templates` вернул пустой массив → фронтенд создаёт 6 глаголов по умолчанию и отправляет на сервер
- Если пользователь нажал «Сбросить» → `POST /verb-templates/reset`

## Компоненты (frontend)

### Новые компоненты
1. **VerbChips** — ряд чипов для desktop. Принимает `onSelect(verb)`, `activeVerb`.
2. **VerbFab** — FAB-кнопка для mobile с веером глаголов. Принимает `onSelect(verb)`.
3. **VerbSettings** — секция на странице настроек для управления списком.

### Изменяемые компоненты
1. **TaskListView** — интеграция VerbChips (desktop) и VerbFab (mobile). Управление состоянием активного глагола, показ/скрытие поля ввода на mobile.

### Хук
1. **useVerbTemplates** — хук для работы со списком глаголов (чтение из Dexie, CRUD через mutations, синхронизация).

## Адаптивный breakpoint

- `< 768px` → mobile режим (FAB, скрытое поле ввода)
- `>= 768px` → desktop режим (чипы по фокусу)

Tailwind: `md:` prefix для desktop, default для mobile.

## Ограничения

- Максимальная длина текста глагола — 30 символов
- Максимум 20 глаголов в списке
- Глагол не может быть пустым
- Конфликт при синхронизации: серверная версия побеждает (last-write-wins)
