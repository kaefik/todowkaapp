# Управление регистрацией пользователей

## Обзор

Todowka поддерживает ограничение количества пользователей в системе через конфигурацию. Это полезно для:
- Персональных приложений с ограниченным доступом
- Командных проектов с фиксированным количеством участников
- Премиум-версий с ограничением пользователей

## Конфигурация

### Переменные окружения

Добавьте в `.env` файл в директории `backend/`:

```bash
# Максимальное количество пользователей (null = без ограничений)
MAX_USERS=10
```

### Параметры

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `MAX_USERS` | `int \| null` | `null` | Максимальное количество пользователей. `null` - без ограничений |

## Использование

### Примеры конфигурации

**Без ограничений (по умолчанию):**
```bash
# Не указывайте MAX_USERS или установите в null
MAX_USERS=
```

**Ограничение на 10 пользователей:**
```bash
MAX_USERS=10
```

**Отключение регистрации:**
```bash
REGISTRATION_ENABLED=false
```

**Регистрация по приглашению:**
```bash
REGISTRATION_ENABLED=false
INVITE_CODE=secret123
```

## API

### GET /api/config

Возвращает текущую конфигурацию:

```json
{
  "registration_enabled": true,
  "max_users": 10,
  "current_users": 7
}
```

### POST /api/auth/register

При попытке регистрации вернёт ошибку если:
- `registration_enabled = false`
- `current_users >= max_users`
- `invite_code` задан и не совпадает

```json
{
  "detail": "Maximum number of users (10) reached"
}
```

## Пользовательский интерфейс

На странице регистрации показывается информация о доступных местах:

```
Доступно мест для регистрации: 3 из 10
```

Если лимит исчерпан, сообщение не показывается, и регистрация будет отклонена сервером.

## Безопасность

- Проверка выполняется на бэкенде перед созданием пользователя
- Невозможно обойти ограничение через API
- Изменение настроек требует перезапуска бэкенда

## Перезапуск после изменения настроек

После изменения `.env` файла необходимо перезапустить бэкенд:

```bash
cd backend
./run.sh
```

Или если запущено вручную:

```bash
# Остановить процесс (Ctrl+C)
# Затем запустить снова
python -m uvicorn app.main:app --reload
```

## Комбинация с другими настройками

### Пример: Пригласительная система с ограничением

```bash
REGISTRATION_ENABLED=true
INVITE_CODE=team2024
MAX_USERS=50
```

Результат:
- Регистрация открыта
- Требуется код приглашения "team2024"
- Максимум 50 пользователей

### Пример: Приватная система для семьи

```bash
REGISTRATION_ENABLED=false
INVITE_CODE=family-secret
MAX_USERS=5
```

Результат:
- Регистрация закрыта публично
- Только с кодом "family-secret"
- Максимум 5 членов семьи

## Мониторинг

Проверить текущее состояние можно через:
- API: `GET /api/config`
- UI: На странице регистрации
- База данных: `SELECT COUNT(*) FROM users`

## Troubleshooting

**Регистрация не работает после настройки:**
1. Проверьте что `.env` файл существует и корректен
2. Убедитесь что бэкенд перезапущен после изменений
3. Проверьте логи бэкенда на наличие ошибок

**Пользователи не видят ограничения:**
- Настройки корректно применены на бэкенде
- API возвращает правильные значения
- UI автоматически обновляет информацию
