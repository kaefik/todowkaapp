### L6-01 — Добавить лимит переподключений SSE

**Goal:** Реализовать счетчик неудачных попыток подключения и остановку переподключений после N попыток.
**Input:** Файл `frontend/src/services/sseManager.ts`
**Output:** Обновленный `frontend/src/services/sseManager.ts` с лимитом переподключений
**Done when:** После 5 неудачных попыток SSE перестает пытаться переподключиться и логирует финальную ошибку.
**Acceptance criteria:**
- [ ] Добавлен счетчик неудачных попыток `reconnectAttempts: number = 0`
- [ ] Максимальное количество попыток задано константой (например, 5)
- [ ] При каждой неудачной попытке счетчик увеличивается
- [ ] При достижении лимита переподключения прекращаются
- [ ] При достижении лимита логируется финальная ошибка с деталями
- [ ] Добавлен метод для ручного сброса счетчика (для ручного переподключения)
**depends_on:** [L5/02]
**impact:** 4 (улучшает UX, предотвращает бесконечный цикл)
**complexity:** 2 (простая логика счетчика)
**risk:** 1 (безопасно)
**priority_score:** (4 × 2 + 1) / 2 = 4.5
**Est. effort:** S (1h)
**LLM Prompt Hint:** В frontend/src/services/sseManager.ts:
1. Добавьте поле reconnectAttempts: number = 0
2. Добавьте константу MAX_RECONNECT_ATTEMPTS = 5
3. В обработчике ошибки SSE увеличивайте счетчик
4. Если reconnectAttempts >= MAX_RECONNECT_ATTEMPTS: прекратите переподключения и логируйте console.error('SSE reconnect limit reached')
5. Добавьте метод resetReconnectAttempts() для сброса счетчика
Убедитесь, что логика работает с существующим механизмом setTimeout для задержки.
