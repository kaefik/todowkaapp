# План улучшений приложения Todowka

## Обзор

7 фич разбиты на 3 этапа по сложности реализации.

---

## Этап 1: Простые фичи (1-2 дня)

### 1. Quick Actions (долгое нажатие)

**Описание:** Меню быстрых действий при долгом нажатии на задачу.

**Действия в меню:**
- Выполнить задачу ✓
- Редактировать ✏️
- Изменить дату 📅
- Переместить в проект 📁
- Удалить 🗑️

**Реализация:**
- Компонент `TaskQuickActions.tsx`
- Интеграция в `TaskListView.tsx`
- Long-press detection (300ms)
- UI: popover с иконками и названиями

**Файлы:**
- `frontend/src/components/TaskQuickActions.tsx` (новый)
- `frontend/src/components/TaskListView.tsx` (изменение)
- `frontend/src/i18n/locales/ru/tasks.json` (добавить ключи)

---

### 2. Пустые состояния (Empty States)

**Описание:** Полезные подсказки в пустых разделах.

**Разделы с подсказками:**
- Inbox: "Добавьте свою первую задачу", "Используйте глаголы для быстрого создания"
- Projects: "Создайте проект для группировки задач"
- Areas: "Области помогают разделить жизнь на сферы ответственности"
- Contexts: "Контексты — места или инструменты для выполнения"
- Tags: "Теги помогают категоризировать задачи"
- Waiting For: "Добавьте задачи, где вы ждёте ответа"
- Someday: "Место для задач на будущее"

**Реализация:**
- Компонент `EmptyState.tsx` (универсальный)
- Props: title, description, actionLabel, onAction
- Интеграция во все списки

**Файлы:**
- `frontend/src/components/EmptyState.tsx` (новый)
- `frontend/src/routes/GtdTaskList.tsx` (изменение)
- `frontend/src/routes/Projects.tsx`
- `frontend/src/routes/Areas.tsx`
- `frontend/src/routes/Contexts.tsx`
- `frontend/src/routes/Tags.tsx`

---

### 3. Интерактивная справка (Walkthrough)

**Описание:** One-time подсказки при первом входе в разделы.

**Поведение:**
- Показывать один раз при первом входе
- Крестик для закрытия
- Кнопка "Больше не показывать"

**Разделы с подсказками:**
- Inbox: "Здесь собираются все новые задачи"
- Tasks: "Полный список всех задач"
- Projects: "Проекты группируют связанные задачи"
- Settings: "Настройте приложение под себя"

**Реализация:**
- Хук `useWalkthrough(stepKey)`
- Store в localStorage: `{ shown: { section: true/false } }`
- Компонент `WalkthroughTooltip.tsx`

**Файлы:**
- `frontend/src/hooks/useWalkthrough.ts` (новый)
- `frontend/src/components/WalkthroughTooltip.tsx` (новый)
- `frontend/src/components/AppLayout.tsx` (интеграция)

## Этап 2: Средние фичи (2-4 дня)

### 4. Swipe-жесты (мобильные)

**Описание:** Свайпы для быстрого выполнения/откладывания задач.

**Поведение:**
- Влево свайп = выполнить задачу
- Вправо свайп = отложить на завтра (due_date = tomorrow)
- Threshold: 100px для срабатывания
- Анимация свайпа

**Реализация:**
- Библиотека: `@dnd-kit/core` (есть) или `react-swipeable`
- Компонент `SwipeableTask.tsx`
- Интеграция в `TaskListView` для мобильных (< 768px)

**Файлы:**
- `frontend/src/components/SwipeableTask.tsx` (новый)
- `frontend/src/components/TaskListView.tsx` (изменение)
- `frontend/src/hooks/useTasks.ts` (методы для postpone)

**Бэкенд:**
- API `PATCH /api/tasks/{id}/postpone` (изменить due_date на завтра)

---

### 5. Daily Review (отдельный раздел)

**Описание:** Раздел для ежедневного обзора задач.

**Структура:**
- Вчера выполненные (краткий список)
- Сегодняшние задачи (active + overdue)
- Предложения:
  - Задачи без проекта
  - Забытые waiting-for (нет активности > 7 дней)
  - Someday задачи, которые пора активировать

**UI:**
- Карточный layout
- Кнопки действий (выполнить, активировать)
- Миникарта с прогрессом

**Реализация:**
- Роут `/review`
- Компонент `DailyReview.tsx`
- API: существующий `/api/review/status`

**Файлы:**
- `frontend/src/routes/DailyReview.tsx` (новый)
- `frontend/src/router.tsx` (добавить роут)
- `frontend/src/components/AppLayout.tsx` (добавить в sidebar)
- `frontend/src/i18n/locales/ru/review.json` (новый файл)

---

### 6. Сплит-скрин на десктопе

**Описание:** Панель просмотра задачи справа (slide-over).

**Поведение:**
- Ширина: 1/3 экрана (33%)
- Открывается при клике на задачу (режим просмотра)
- Кнопка "Редактировать" открывает полную модалку
- Закрытие: клик вне панели или ESC

**UI:**
- Заголовок задачи
- Описание
- Проект, область, контекст, теги
- Дедлайн и время
- Подзадачи
- Кнопка "Редактировать"

**Реализация:**
- Компонент `TaskSlideOver.tsx`
- State в `TaskListView`: `openSlideOver(taskId)`
- Close по `onClose` и `useEffect` для ESC

**Файлы:**
- `frontend/src/components/TaskSlideOver.tsx` (новый)
- `frontend/src/components/TaskListView.tsx` (изменение)
- `frontend/src/components/TaskDetailModal.tsx` (использовать общий компонент)

## Этап 3: Сложные фичи (4-7 дней)

### 7. Calendar View

**Описание:** Календарное отображение задач.

**Виды отображения:**
- День: часовые блоки, задачи как карточки
- Неделя: дневные колонки, задачи строками
- Месяц: сетка дней, мини-карточки

**Данные:**
- Все задачи (не только с дедлайном)
- Без дедлайна — показывать в "Без даты" секции
- Цветовое кодирование по проекту

**UI:**
- Хедер: текущая дата, переключатель вида (день/неделя/месяц), навигация
- Тело: сетка календаря
- Клик на задачу → открыть детали
- Клик на день → создать задачу с дедлайном

**Реализация:**
- Библиотека: `@fullcalendar/react` или `react-big-calendar`
- Компонент `CalendarView.tsx`
- Роут `/calendar`

**Файлы:**
- `frontend/src/routes/Calendar.tsx` (новый)
- `frontend/src/components/CalendarView.tsx` (новый)
- `frontend/src/router.tsx` (добавить роут)
- `frontend/src/components/AppLayout.tsx` (добавить в sidebar)
- `frontend/src/i18n/locales/ru/calendar.json` (новый файл)
