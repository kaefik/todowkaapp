Тебе нужно добавить новую запись в файл BACKLOG.md проекта.

**Категория записи:** $1
- Если указано "idea" или "идея" — добавить в секцию "## Идеи и возможности"
- Если указано "bug" или "баг" — добавить в секцию "## Баги"
- Если указано "test" или "тест" — добавить в секцию "## Запланированные тесты" (если её нет — создать после секции "## Баги")

**Описание записи:** $2

## Правила

1. Прочитай текущее содержимое файла @BACKLOG.md
2. Определи категорию из первого аргумента ($1):
   - idea/идея → секция "## Идеи и возможности"
   - bug/баг → секция "## Баги"
   - test/тест → секция "## Запланированные тесты"
3. Добавь запись в конец соответствующей секции в формате:
   - Для идей: `- **Название**: Описание`
   - Для багов: `- **Описание бага**: Шаги воспроизведения / контекст`
   - Для тестов: `- **Что тестируем**: Описание проверяемого сценария`
4. НЕ изменяй существующие записи
5. Сохрани файл

Примеры использования:
- /backlog idea "Автоочистка корзины" "Удалять задачи из корзины через 30 дней"
- /backlog bug "Фильтры не сбрасываются" "При переходе на главную фильтры из localStorage применяются"
- /backlog test "Экспорт задач" "Проверить что JSON экспорт содержит все связанные данные"
